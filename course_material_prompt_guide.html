Step 1 — Course Setup Prompt (run this first, once per course)
Setup Course Context Block
Paste this once to establish your course profile before generating any session material.
You are a professional academic study material designer. I am a university lecturer creating teaching materials for bachelor-level students. Here is my course profile: COURSE NAME: [e.g. Introduction to Marketing] DEPARTMENT: [e.g. Business Administration] STUDENT LEVEL: Bachelor's Year [1/2/3/4] SESSION DURATION: 50 minutes per class TOTAL WEEKS: [e.g. 14 weeks] SESSIONS PER WEEK: [e.g. 2] STUDENT BACKGROUND: [e.g. No prior marketing knowledge, basic business understanding] LEARNING STYLE: Mix of theory, real-world examples, and interactive elements TONE: Professional but engaging; friendly academic style OUTPUT FORMAT: [PPT slides / PDF handout — choose one] Remember this context for all future material generation in this chat.
Step 2 — Per-Session Prompt (run once per lecture)
Per Session Lecture Material Generator
Fill in the brackets and send this for each session. This is your core workhorse prompt.
Generate complete study material for the following session: SESSION: Week [X], Lecture [Y] TOPIC: [e.g. Consumer Behaviour and Decision Making] ALLOCATED HOURS: [e.g. 1.5 hours = 3 × 50-min sessions] SUBTOPICS TO COVER: 1. [Subtopic A] 2. [Subtopic B] 3. [Subtopic C] LEARNING OUTCOMES: By end of this session, students will be able to: - [Outcome 1] - [Outcome 2] - [Outcome 3] STRUCTURE THE MATERIAL AS FOLLOWS: 🎯 OPENING (5 min) - Hook: A surprising fact, statistic, or question to grab attention - Brief agenda preview 📚 CONTENT SECTIONS (35 min) For EACH subtopic include: • Clear definition in plain English • Core theory or concept (2–3 bullet points max per slide/section) • Real-world example from a well-known company or everyday life • Local/regional example relevant to [your country/context] • Mini case study (2–3 sentence scenario students can discuss) • Common misconception students have — and the correction • Fun element: analogy, mnemonic, or pop culture connection 🧠 CHECK FOR UNDERSTANDING (5 min) - 2 quick quiz questions (MCQ format) with answers - 1 discussion prompt for class participation 💡 REAL-LIFE APPLICATION (3 min) - "This matters because..." — direct link to future careers or daily life 📝 SUMMARY & TAKEAWAYS (2 min) - 3 key points to remember - Preview of next session VISUAL/DESIGN NOTES: - Suggest where to add diagrams, charts, or images - Mark slides that benefit from a visual vs text-heavy - Add speaker notes for the lecturer under each slide TONE: Engaging, clear, not condescending. Use "we" and "you" to address students directly. Avoid jargon unless defining it.
Step 3 — Optional Add-ons
Student Handout Version
Now convert the above into a student handout PDF. Remove speaker notes. Add blank "Notes" space after each section. Include a glossary of new terms at the end. Add 3 self-study questions students can review at home.
Quiz / Assessment
Create a 10-question quiz for this session's content. Include: 5 MCQ, 3 true/false, 2 short answer. Vary difficulty (easy/medium/hard). Provide an answer key with brief explanations.
Cost & effort comparison
PDF / Markdown
Cheapest — Claude generates raw text/markdown; convert with any tool
Fastest edits — find & replace text, no layout to break
Universal — opens on any device, prints cleanly
No software needed beyond a browser
Less visual — harder to present in-class
Static — no animations or builds
Setup time: ~5 min per session
PowerPoint (.pptx) Recommended
Best for teaching — visual flow, slide-by-slide control
Speaker notes built-in — your cue cards are inside the file
Can export as PDF for student handout (best of both)
Reusable across semesters — just update content
Higher token cost — Claude must generate layout code
More complex edits if redesigning layout
Setup time: ~15–25 min per session
Recommended approach: Generate content as PDF/markdown first to verify accuracy and flow cheaply. Then ask Claude to convert the approved content into a PPT. This gives you fast review + professional delivery output — best of both.
Token cost reality check
Markdown/text response → 1× base cost
PDF (via Python script) → 1.2× — adds formatting layer
PPT (pptxgenjs code) → 2–3× — layout code adds significant tokens
PPT with custom design + images → 3–5× — most expensive
Scenario A — Minor text change (typo, update a fact)
In [Session 3 material], on the slide/section about [topic], change "[old text]" to "[new text]". Keep everything else identical.
For PPT files: Open in PowerPoint → Ctrl+H (Find & Replace) → done in 30 seconds. No need to regenerate.
Scenario B — Add a new example or case study
For Session [X], topic "[Subtopic]", add one more real-world example. The example should: be from [region/industry], be relatable to 20-year-old students, and follow the same format as the existing examples (situation → action → result).
Scenario C — Restructure for different time allocation
The original Session [X] was planned for [90 min / 2 sessions]. I now only have [50 min / 1 session]. Please condense the material. Keep: [list must-keep items]. Cut or move to self-study: [less critical parts]. Adjust the opening/closing timings accordingly.
Scenario D — Regenerate one section only
Keep all of Session [X] the same except for the [quiz / case study / summary]. Regenerate only that part. Make it more [challenging / simpler / industry-specific].
Scenario E — Translate or localize for new audience
Take Session [X] material. Replace all Western examples with examples from [Nepal / South Asia / your country]. Keep the theory and structure identical. Ensure all company names and scenarios are locally recognizable to students in [your city/region].
Scenario F — Convert between formats
Take the Session [X] content (pasted below) and convert it into [PPT slides / a student PDF handout / a one-page summary sheet]. Maintain all content but optimize the layout for [presentation / self-study / quick reference]. [paste your content here]
Workflow recommendation
1
Send Setup Prompt once — establishes course context Claude will remember throughout the conversation.
2
Generate as markdown first — review accuracy, add your own notes, correct facts. Much faster to edit at this stage.
3
Convert to PPT/PDF after approval — only convert once you're happy with the content, to avoid wasted tokens on redesigns.
4
Save your prompts — keep the per-session prompt in a doc. Just fill in the brackets for each new session.
5
Build a template PPT once — your colors, fonts, logo. Then ask Claude to fill content into it. Never redesign from scratch again.
Quality boosters to add to any prompt
Add "Use the Feynman technique — explain it like I'm explaining to a smart 18-year-old who has never heard this term."
Add "For every definition, give one analogy to something from everyday life (food, sport, social media, money)."
Add "Flag any slide where students are likely to get confused and suggest a 60-second activity to re-engage them."
Add "Where relevant, suggest a 2-minute YouTube video topic I can search for to illustrate this concept visually."
Add "Mark slides as [THEORY], [EXAMPLE], [ACTIVITY], or [QUIZ] so I can pace the lecture easily."
Common mistakes to avoid
Don't paste all your topics into one prompt — one session at a time gives better quality and lower cost
Don't skip the Setup Prompt — without context, Claude defaults to generic content
Don't ask Claude to make PPT on first try — approve content first, then format
Don't forget to specify your students' background — huge impact on explanation depth
